PODSUMOWANIE PROJEKTU:
Tworzenie kompleksowego pipeline'u Machine Learning do przewidywania sukcesu gier 
na platformie Steam na podstawie zestawu danych zawierającego cechy gier 
(cena, recenzje, oceny, platformy, itp.). 

OPARTY NA DANYCH: Dataset z Kaggle zawierający ~122k gier Steam
CEL: Predykcja czy gra będzie "Sukces" (Wysoka Ocena) czy "Porażka" (Niska/Średnia Ocena)
CEL 2: Porównanie 6 algorytmów ML oraz wizualizacja wyników

5.4. Tworzenie cech ocen i DEFINICJA SUKCESU
    FUNKCJA: FeatureEngineer.create_score_features()
    
    NOWA DEFINICJA SUKCESU (zmieniona z oryginalnej):
    ┌──────────────────────────────────────────────────────────┐
    │ Is_highly_rated = (Review_ratio >= 0.70 AND             │
    │                    Total_reviews >= 20)                 │
    │                OR (Metacritic_score >= 75)              │
    │                                                          │
    │ WYJAŚNIENIE:                                             │
    │ - Gra ma co najmniej 70% pozytywnych recenzji            │
    │   I więcej niż 20 recenzji                              │
    │ LUB                                                      │
    │ - Gra ma ocenę Metacritic >= 75 (Good/Excellent)        │
    │                                                          │
    │ STARZA DEFINICJA (zbyt restrykcyjna):                    │
    │ Is_highly_rated = (Review_ratio >= 0.80 AND             │
    │                    Total_reviews >= 30)                 │
    │ -> Rezultat: 99.9% klasy 0, 0.1% klasy 1                │
    │ -> Problem: Niezbilansowana (1 tys razy różnica)         │
    │                                                          │
    │ NOWA DEFINICJA (zbilansowana):                           │
    │ -> Rezultat: ~60-65% klasy 0, ~35-40% klasy 1           │
    │ -> Rozsądne dla ML                                       │
    └──────────────────────────────────────────────────────────┘
    
    DODATKOWE CECHY:
    ├── Has_metacritic      = Czy ma ocenę Metacritic (0/1)
    ├── Metacritic_category = Kategoryzacja oceny (Poor/Fair/Good/Excellent)
    └── User_score_normalized = Ocena użytkownika / 10 (0-1)

6.2. Detekcja outlierów
    FUNKCJA: DataValidator.detect_outliers()
    METODA: IQR (Interquartile Range)
    
    def detect_outliers(col):
        Q1 = col.quantile(0.25)
        Q3 = col.quantile(0.75)
        IQR = Q3 - Q1
        lower_bound = Q1 - 1.5 * IQR
        upper_bound = Q3 + 1.5 * IQR
        outliers = (col < lower_bound) | (col > upper_bound)
        return outliers
    
    PRZYKŁAD:
    - Price: [0, 999.98] -> outliery: wartości > 100 (5% danych)
    - Log_owners: [-1, 15] -> ~2% outlierów

9.3. Trenowanie modeli
    FUNKCJA: AdvancedModelTrainer.train_and_evaluate_models()
    
    DLA KAŻDEGO MODELU:
    1. Pomiar czasu startu: start_time = time.time()
    2. Trenowanie:
       model.fit(X_train, y_train, sample_weight=sample_weights)
    3. Predykcja: y_pred = model.predict(X_test)
    4. Prawdopodobieństwa: y_proba = model.predict_proba(X_test)[:, 1]

9.4. Metryki ewaluacji
    FUNKCJA: Obliczane dla każdego modelu
    BIBLIOTEKA: sklearn.metrics
    
    METRYKI (dla każdego modelu):
    
    1. ACCURACY (Dokładność)
       accuracy = (TP + TN) / (TP + TN + FP + FN)
       Interpretacja: % poprawnych predykcji
       Wada: Nie działa dobrze dla niezbalansowanych klas
    
    2. PRECISION (Precyzja)
       precision = TP / (TP + FP)
       Interpretacja: % predykcji pozytywnych, które są prawdziwe
       Znaczenie: Uniknie fałszywych alarmów
    
    3. RECALL (Czułość)
       recall = TP / (TP + FN)
       Interpretacja: % rzeczywistych pozytywnych znalezionych
       Znaczenie: Złapanie maksimum pozytywnych
    
    4. F1-SCORE
       f1 = 2 * (precision * recall) / (precision + recall)
       Interpretacja: Średnia harmoniczna precision i recall
       Najlepiej dla niezbalansowanych danych
    
    5. ROC-AUC (Area Under ROC Curve)
       roc_auc = auc(fpr, tpr)
       fpr = False Positive Rate
       tpr = True Positive Rate
       Interpretacja: Prawdopodobieństwo że model będzie lepszy niż random
       Zakres: [0, 1], gdzie 1 = idealne, 0.5 = random
    
    6. CZAS TRENOWANIA
       training_time = end_time - start_time (w sekundach)
       Znaczenie: Efektywność obliczeniowa
    
    WYNIKI REPREZENTATYWNE (przykład):
    ┌─────────────────────────────────────────────────────────┐
    │ Model              │ Acc  │ Prec │ Rec  │ F1  │ ROC-AUC │
    ├─────────────────────────────────────────────────────────┤
    │ Logistic Regr.    │ 0.75 │ 0.62 │ 0.58 │0.60 │ 0.82    │
    │ Decision Tree     │ 0.72 │ 0.58 │ 0.48 │0.52 │ 0.75    │
    │ Random Forest     │ 0.81 │ 0.71 │ 0.65 │0.68 │ 0.88    │
    │ LightGBM         │ 0.84 │ 0.76 │ 0.72 │0.74 │ 0.91    │
    │ XGBoost          │ 0.85 │ 0.77 │ 0.74 │0.75 │ 0.92    │
    │ Neural Network   │ 0.80 │ 0.68 │ 0.61 │0.64 │ 0.86    │
    └─────────────────────────────────────────────────────────┘

    - FN (czerwone): Model fałszywie przewidział "negatywne"

10.3. Krzywe ROC i Precision-Recall
    FUNKCJA: AdvancedModelEvaluator.plot_roc_and_pr_curves()
    BIBLIOTEKA: sklearn.metrics.roc_curve, precision_recall_curve
    
    KRZYWA ROC:
    - X-oś: False Positive Rate (% fałszywych alarmów)
    - Y-oś: True Positive Rate (% złapanych pozytywnych)
    - Pole pod krzywą (AUC): Jego wielkość = jakość modelu
    - AUC = 0.5  -> Model jest losowy (nie godny)
    - AUC = 0.92 -> Model jest bardzo dobry (88% między idealnym a losowym)
    
    KRZYWA PRECISION-RECALL:
    - X-oś: Recall (czułość)
    - Y-oś: Precision (precyzja)
    - Przydatna dla niezbalansowanych danych
    - AUC-PR = pole pod tą krzywą
    
    OUTPUT: roc_pr_curves_advanced.png
